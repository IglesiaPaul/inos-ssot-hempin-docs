## Application Checklist

Use this checklist for each grant application:

**Phase 1: Eligibility (Week 1)**
- [ ] Eligibility criteria confirmed
- [ ] Partner requirements met (if consortium)
- [ ] Budget ceiling/floor checked
- [ ] Geographic eligibility confirmed
- [ ] Topic area alignment verified
- [ ] Go/No-Go decision made

**Phase 2: Preparation (Week 2-4)**
- [ ] Required documents gathered
- [ ] Partnership letters secured
- [ ] Technical narrative outline created
- [ ] Budget template downloaded
- [ ] Evaluation criteria studied
- [ ] Writer/editor assigned

**Phase 3: Drafting (Week 4-8)**
- [ ] Technical narrative drafted
- [ ] Impact section completed
- [ ] Budget justification written
- [ ] CVs/bios compiled
- [ ] Letters of commitment collected
- [ ] All attachments prepared

**Phase 4: Review (Week 8-10)**
- [ ] Internal review complete
- [ ] External review (if applicable)
- [ ] Compliance check passed
- [ ] Budget review complete
- [ ] Final edits incorporated
- [ ] All files formatted correctly

**Phase 5: Submission (Week 10-12)**
- [ ] Portal registration complete
- [ ] All sections filled
- [ ] All attachments uploaded
- [ ] Submission confirmed
- [ ] Confirmation email received
- [ ] Tracking number recorded

**Phase 6: Follow-Up (Post-Submission)**
- [ ] Acknowledgment received
- [ ] Evaluation timeline noted
- [ ] Interview preparation (if applicable)
- [ ] Grant agreement review (if awarded)
- [ ] Reporting system setup (if awarded)

## Grant Writing Best Practices

### Do's

| Do | Why |
|----|-----|
| Start early (12 weeks before deadline) | Quality requires time |
| Study evaluation criteria thoroughly | Write to the rubric |
| Use clear, jargon-free language | Evaluators are not always experts |
| Include measurable impact metrics | Quantifiable outcomes win |
| Show consortium strength (if applicable) | Collaboration is valued |
| Align with EU/UN priorities | Policy alignment matters |
| Budget realistically | Unrealistic budgets get cut |
| Proofread multiple times | Errors signal carelessness |

### Don'ts

| Don't | Why |
|-------|-----|
| Wait until the last week | Rushed applications fail |
| Use excessive technical jargon | Evaluators may not understand |
| Overpromise on deliverables | Unachievable goals hurt credibility |
| Copy-paste from previous applications | Each grant is unique |
| Ignore page limits | Non-compliance = rejection |
| Submit without compliance check | Administrative rejection is common |
| Forget to budget for dissemination | Communication is required |
| Neglect the impact section | Impact is heavily weighted |

---

## Common Rejection Reasons

| Reason | Prevention |
|--------|------------|
| Not aligned with call topic | Study work programme carefully |
| Weak impact section | Use theory of change, measurable metrics |
| Unrealistic budget | Benchmark against similar projects |
| Weak consortium | Add credible partners early |
| Poor writing quality | Hire professional grant writer |
| Missing eligibility criteria | Complete eligibility checklist |
| Late submission | Submit 24+ hours early |
| Administrative errors | Use compliance checklist |

## Post-Award Management

### Reporting Requirements

| Report Type | Frequency | Content |
|-------------|-----------|---------|
| Progress Report | Every 6-12 months | Milestones, deliverables, challenges |
| Financial Report | Every 6-12 months | Budget vs. actual, eligible costs |
| Impact Report | Annual | KPIs, outcomes, beneficiaries |
| Final Report | End of project | All deliverables, financial summary |
| Audit Report | As required | External audit of costs |
