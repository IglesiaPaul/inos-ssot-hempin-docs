📦 Entity Layer

  • Canonical representation of every verified actor (farmers, processors, brands, regulators)
  • Unique identifiers with verification status and provenance trail
  • Attributes: Location, certifications, capacity, compliance status
  • Dynamic updates: Event-sourced for immutable audit trail

🔗 Relationship Layer

  • Every interaction between entities (trades, contracts, certifications, samples)
  • Time-series events: Immutable record of all relationships
  • Graph traversals: Enable complex queries (e.g., "Show all suppliers of Supplier X")
  • Verification status: Each relationship has confidence score and verification level

⚖️ Compliance Layer

  • 57 jurisdictions encoded as machine-readable graph constraints (SHACL/Cypher)
  • Real-time validation: Transactions blocked if non-compliant
  • Regulatory ontology: THC limits, licensing, labeling rules per jurisdiction
  • Iterative expansion: Start with 5 jurisdictions (TH, FR, DE, NL, AU), expand to 57+

💎 Value Layer

  • Economic flow graph: Transactions, fees, Regeneration Fund contributions
  • 30% automatic routing: Transaction value flows to environmental cleanup
  • Transparent impact metrics: Link every transaction to verified cleanup projects
  • NADA token economics: Future token layer for governance and incentives

🤖 AI Robustness Measures (EIC Requirement)

  • Technical Robustness: Confidence scoring (>0.85 threshold) + human-in-the-loop validation for low-confidence extractions
  • Social Robustness: Training data diversified across languages (EN, FR, TH, RO) to reduce bias; human review for sensitive regulatory decisions
  • Reliability: Service-level agreements (99.9% uptime); error logging and incident response protocols
  • Explainability: Each graph node includes provenance trail (source text, confidence score, reviewer ID) for auditability
